# Yandex Maps Parser

Веб-приложение для парсинга данных организации из Яндекс.Карт, получения рейтинга, количества оценок, количества отзывов и полного списка доступных отзывов без использования официального API.

## Что умеет приложение

- Авторизация через Laravel Sanctum.
- Один тестовый пользователь без регистрации.
- Сохранение ссылки на организацию из Яндекс.Карт.
- Асинхронный запуск парсинга через очередь.
- Получение данных организации:
  - название;
  - средний рейтинг;
  - количество оценок;
  - количество отзывов.
- Сбор всех доступных отзывов, которые удаётся получить через парсер.
- Сохранение отзывов в базу данных.
- Повторный запуск парсинга: старые отзывы удаляются, загружаются новые актуальные данные.
- Пагинация отзывов по 50 записей.
- История запусков парсинга.
- Отображение статуса задачи и прогресса.

## Важное про проект

Это рабочий прототип, который рассчитан на дальнейшее расширение.

Система уже подготовлена так, чтобы в ней могло быть больше одного пользователя, но на текущем этапе поставляется только один демонстрационный аккаунт. Регистрация отключена намеренно.

Парсинг реализован через отдельный Node.js-скрипт с Puppeteer. Такой подход выбран потому, что у Яндекс.Карт нет публичного API для полного списка отзывов, а данные подгружаются динамически при прокрутке страницы.

## Установка проекта

### 1. Склонируйте репозиторий

```powershell

```

```powershell
Copy-Item .env.example .env
```

```powershell

```

## Настройка `.env`

Ниже пример основных параметров. Значения для своей среды нужно заменить вручную.

```env
APP_NAME=Laravel
APP_ENV=local
APP_KEY=base64:YOUR_APP_KEY_HERE
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=yandex_parser
DB_USERNAME=password
DB_PASSWORD=password

SESSION_DRIVER=database
QUEUE_CONNECTION=database
CACHE_STORE=database
FILESYSTEM_DISK=local
```

### Что важно проверить отдельно

- `DB_HOST` должен указывать на ваш сервер .
- `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD` должны совпадать с вашей локальной или серверной БД.
- `QUEUE_CONNECTION=database` — очередь работает через таблицу jobs.
- `SESSION_DRIVER=database` — сессии хранятся в БД.
- `CACHE_STORE=database` — кэш также использует БД.

## Как устроен парсинг

Парсинг вынесен из контроллеров в отдельный сервис и фоновую задачу.

### Основные части

- `App\Services\YandexMapsParserService` — сервис-обёртка, который запускает внешний парсер.
- `App\Jobs\SyncYandexPlaceJob` — очередь, которая выполняет синхронизацию организации и отзывов.
- `scripts/yandex-parser.cjs` — внешний Node.js-скрипт, который открывает страницу организации через headless Chromium и собирает данные.

### Почему выбран Puppeteer

У Яндекс.Карт нет удобного публичного API для этой задачи. Отзывы подгружаются динамически и требуют прокрутки страницы, поэтому headless-браузер оказался самым надёжным решением для текущего этапа.

### Логика сохранения данных

1. Пользователь сохраняет ссылку на организацию.
2. Laravel отправляет задачу в очередь.
3. Job запускает Node.js-парсер.
4. Парсер получает данные организации и список отзывов.
5. Данные сохраняются в БД.
6. При повторном запуске старые отзывы удаляются и загружаеться новый актуальный набор данных.
7. В раздел "История" заноситься информация о выполненных запросах по организациям.

## Пагинация

Отзывы не загружаются по 50 штук с Яндекса на каждый запрос интерфейса. Сначала они сохраняются в БД, а дальше фронтенд получает их постранично из backend.

Такой подход уменьшает количество обращений к внешнему источнику и делает работу интерфейса стабильнее.

## Production-запуск

Для production используется та же логика, что и локально, но с постоянным запуском очереди и собранным фронтендом.

### Сборка фронтенда

```powershell
npm run build
```

### Очередь в production

Рекомендуется запускать worker через Supervisor.

Пример конфигурации:

```ini
[program:yandex-parser-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/yandex-parser/artisan queue:work --sleep=3 --tries=3
autostart=true
autorestart=true
numprocs=1
redirect_stderr=true
stdout_logfile=/var/www/yandex-parser/storage/logs/worker.log
```

## Docker

Изначально планировалась полная Docker-сборка с `docker-compose`, но на текущем этапе проект был завершён без контейнеризации из-за ограничений по времени.

Поэтому локальный запуск сейчас выполняется стандартными средствами PHP, Node.js и MySQL.

## Известные ограничения

- Регистрация отсутствует: доступен только один тестовый пользователь.
- Docker-сборка не добавлена только из за ограничения по времени.
- Проект рассчитан на расширение, но текущая версия закрывает полностью основной функционал .

## Что можно улучшить, если будет больше времени

- Добавить полноценный `docker-compose` для backend, frontend, queue и database.
- Вынести деплой в готовый production pipeline.
- Добавить WebSocket-обновление прогресса без опроса.
- Улучшить восстановление после временных ошибок парсера.
- Добавить повторные автоматические попытки для нестабильных запросов.
- Расширить пользовательские роли и управление доступом.
- Добавить более подробные логи и мониторинг фоновых задач.

## Подход к надёжности данных

- Все доступные отзывы сохраняются в БД.
- При повторном парсинге старые отзывы очищаются, чтобы данные не дублировались и пользователь получал именно актуальный набор данных .
- Пагинация выполняется по 50 отзывов.
- Счётчики и рейтинг сохраняются и отображаются из актуальных данных парсинга в разделе "История".
